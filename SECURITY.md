# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 0.4.x   | ✅        |
| < 0.4   | ❌        |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

If you discover a vulnerability, please send an email to the maintainer via the
contact information listed on the [GitHub profile](https://github.com/abhishekgautam95).

Include as much of the following as possible:
- Type of vulnerability (e.g. injection, path traversal, secret exposure)
- File(s) and line number(s) of the affected source code
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Potential impact of the vulnerability

You should receive an acknowledgement within **48 hours** and a full response
within **7 days** indicating next steps. If you have not received a reply after
7 days, please follow up.

## Responsible Disclosure

We follow a **coordinated disclosure** policy. Please give us reasonable time to
address the issue before making it public. We will credit you in the release
notes if you wish.
